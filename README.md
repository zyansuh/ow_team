# 티어맞춤 팀짜기

오버워치 커스텀·서버 내전을 위한 **역할별 티어 밸런스 팀 편성** 웹앱입니다.  
**5:5 / 6:6 모드**를 고르고 닉네임·포지션별 티어를 등록하면, 역할 실력을 맞춰 팀을 나눈 뒤 싱글 엘리미네이션 토너먼트까지 진행할 수 있습니다.

> 대규모 서버(수백 명) 내전을 염두에 두고, **팀원·팀 수 모두 제한 없이** 늘릴 수 있습니다.  
> **5:5** = 탱1·딜2·힐2 / **6:6** = 탱2·딜2·힐2  
> **탱·힐·딜 티어를 모두 적은 뒤 무작위(플렉스)** 를 켜면 **빈 슬롯을 해당 포지션 티어로 자동 채웁니다.**  
> **예비인원** · **팀 새로 짜기** · **미배치/언랭** · **반응형** · **라이트 모던 UI**를 지원합니다.

**저장소:** [https://github.com/zyansuh/ow_team](https://github.com/zyansuh/ow_team)

## 주요 기능

| 기능 | 설명 |
|------|------|
| **5:5 모드** | 팀당 **탱커 1 · 딜러 2 · 힐러 2** (5인) |
| **6:6 모드** | 팀당 **탱커 2 · 딜러 2 · 힐러 2** (6인) |
| 모드 선택 UI | 페이지 상단에서 전환 · 선택값은 `localStorage`에 유지 |
| 팀원 등록 | 닉네임 + 포지션별 티어 · 인원 제한 없음 |
| 멀티 포지션 티어 | 탱/힐/딜마다 다른 티어 입력 |
| 무작위(플렉스) | 탱·힐·딜 티어를 모두 작성한 뒤 켜면 **빈 슬롯 자동 투입** |
| 티어 체계 | **미배치/언랭** + 브론즈~챔피언 × 디비전 1~5 |
| 포지션별 밸런싱 | 전담 배치 → 플렉스 빈자리 → 역할·팀 평균 MMR 균등화 |
| 탱커 정원 가드 | 모드별 탱커 수를 **절대 초과하지 않음** (멀티포지션 우선 재배정) |
| **팀 새로 짜기** | 편성 후 인원을 섞어 **다른 밸런스 편성**을 다시 뽑음 |
| **예비인원** | 닉네임만 등록 · **팀짜기 미포함** · 결과 화면 예비 명단 표시 |
| 예비로 보내기 | 로비 팀원을 예비 칸으로 이동 |
| 팀 수 | 1팀부터 무제한 |
| 토너먼트 | N팀 싱글 엘리미네이션 · 부전승 · 가로 스냅 스크롤 |
| 로컬 저장 | 팀원·예비·모드 `localStorage` · 구버전 데이터 마이그레이션 |
| 반응형 UI | 풀폭 버튼, 터치 영역, 모바일 브래킷 스와이프 |
| **라이트 UI** | 소프트 블루 배경 · 글래스 패널 · pill CTA · 히어로 그라데이션 |

## 모드별 구성

| 모드 | 탱커 | 딜러 | 힐러 | 팀 인원 |
|------|------|------|------|---------|
| **5:5** | 1 | 2 | 2 | 5 |
| **6:6** | 2 | 2 | 2 | 6 |

완편 예: 5:5 → `탱 1/1 · 딜 2/2 · 힐 2/2` / 6:6 → `탱 2/2 · 딜 2/2 · 힐 2/2`

## 팀짜기 흐름 (요약)

1. 상단에서 **5:5 또는 6:6** 선택  
2. 로비에 **출전 인원** 등록 (포지션·티어) · 필요 시 **예비인원** 별도 등록  
3. **전담** 인원을 역할 슬롯에 티어 맞춰 배치 (모드 정원 내)  
4. **무작위(플렉스)** 로 **빈 슬롯**을 해당 포지션 티어로 채움  
5. 남는 인원은 인원·MMR이 적은 팀에 오버플로 (탱커 정원은 넘기지 않음)  
6. 역할·팀 평균 MMR이 고르게 되도록 스왑 미세 조정  
7. 결과가 마음에 안 들면 **팀 새로 짜기**로 다른 편성 재추첨  

예비인원은 4~6단계에 **포함되지 않습니다.**

## 예비인원

| 항목 | 내용 |
|------|------|
| 등록 | 로비 하단 **예비인원** 칸에 닉네임만 입력 |
| 로비 → 예비 | 팀원 카드의 **예비로 보내기** 아이콘 |
| 팀짜기 | 예비는 밸런싱·토너먼트에 **미참여** |
| 결과 표시 | 편성 결과 상단에 **예비 명단** |
| 저장 | `localStorage` 키 `squad-forge-reserves` |

### 알고리즘 (직접 구현)

- **모드별 슬롯 정원** — `MODE_COMPOSITION['5v5'|'6v6']`
- **전담 우선 배치** — 역할 티어 · 총점 낮은 팀 스네이크
- **플렉스 빈 슬롯 투입** — 무작위 → 부족한 탱/딜/힐
- **탱커 캡** — 멀티포지션 우선 딜/힐 재배정 · 전담은 탱 유지
- **역할·총점 스왑** — 역할 MMR + 팀 평균 MMR 균등화
- **팀 새로 짜기** — 입력 셔플 + 동점 랜덤으로 다른 편성
- **N팀 브래킷** — 부전승 · 승자 진출

백엔드·DB 없음. 데이터는 브라우저 `localStorage`만 사용합니다.

```ts

Player {
  id, nickname,
  roles: [{ position, tier }, ...]  // random = 플렉스 표시
}

ReserveEntry { id, nickname }  // 팀짜기 미포함 예비

TeamMember {
  player: Player
  slottedRole: 'tank' | 'healer' | 'dealer'  // 배정된 역할
}

Team { id, name, members: TeamMember[], totalMmr }
```

## 사용 방법

1. 상단에서 **5:5** 또는 **6:6** 선택  
2. 로비에서 닉네임 + 탱/힐/딜 티어 입력 (`미배치/언랭` 가능)  
3. 대기·교체 인원은 **예비인원**에 닉네임만 추가 (또는 로비에서 예비로 보내기)  
4. 플렉스면 **무작위** 켜기 (세 티어 모두 필요)  
5. 팀 수 설정 후 **티어 맞춰 팀짜기**  
6. 결과가 아쉽면 **팀 새로 짜기**로 재편성  
7. 결과에서 배정 역할·완편·예비 명단 확인  
8. 토너먼트 승자 선택 (모바일은 브래킷 좌우 스와이프)

최소 인원(출전): **팀 수 이상**. 예비는 이 수에 포함되지 않습니다.  
전원 완편≈ **팀 수 × 5**(5:5) 또는 **팀 수 × 6**(6:6).

## 티어 MMR (요약)

`미배치/언랭` → `브론즈 5` … → `챔피언 1`

| 항목 | 내용 |
|------|------|
| 미배치/언랭 | MMR `0`, 디비전 없음 |
| 디비전 | 5(낮음) → 1(높음) |
| 팀 합계 | 멤버 **배정 역할** 티어 MMR 합 |
