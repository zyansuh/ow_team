# 티어맞춤 팀짜기

오버워치 커스텀·서버 내전을 위한 **역할별 티어 밸런스 팀 편성** 웹앱입니다.  
닉네임과 **포지션별 티어**를 등록하면 탱커/힐러/딜러끼리 실력이 비슷하도록 팀을 나누고, 원하는 팀 수만큼 싱글 엘리미네이션 토너먼트까지 진행할 수 있습니다.

> 대규모 서버(수백 명) 내전을 염두에 두고, **팀원·팀 수 모두 제한 없이** 늘릴 수 있게 만들었습니다.  
> **한 명이 여러 포지션**을 가질 수 있고, **포지션마다 다른 티어**를 적을 수 있습니다.  
> **모바일·태블릿·데스크톱** 반응형으로, 폰에서도 로비·팀짜기·토너먼트를 편하게 돌릴 수 있습니다.

**저장소:** [https://github.com/zyansuh/ow_team](https://github.com/zyansuh/ow_team)

## 주요 기능

| 기능 | 설명 |
|------|------|
| 팀원 등록 | 닉네임 + 포지션(복수 선택) + **포지션별 티어** · 인원 제한 없음 |
| 멀티 포지션 | 예: 탱커 골드3 · 힐러 플래티넘2처럼 역할마다 티어를 따로 입력 |
| 티어 체계 | 브론즈~챔피언 × 디비전 1~5 (디비전 5가 낮고 1이 높음) |
| 포지션별 밸런싱 | 탱/힐/딜 각각 **해당 포지션 티어**로 스네이크 드래프트 후 미세 조정 |
| 팀 수 | 1팀부터 무제한 (`+/-`, 직접 입력, `+5` / `+10` / `+20`) |
| 토너먼트 | N팀 싱글 엘리미네이션 · 2의 거듭제곱이 아니면 부전승 자동 처리 |
| 로컬 저장 | `localStorage` 유지 · 구버전(단일 포지션) 데이터 자동 마이그레이션 |
| 반응형 UI | 좁은 화면 풀폭 버튼, 스냅 가로 스크롤 브래킷, 터치 친화적 높이 |

### 알고리즘 (직접 구현)

- **멀티포지션 인식 스네이크 드래프트** — 미배정 인원만, 해당 역할 티어로 탱→힐→딜 배정
- **한 사람 = 한 팀** — 여러 포지션이어도 팀에 한 번만 배정
- **동일 포지션 스왑 미세 조정** — 역할별 MMR 편차 최소화 (역할 티어 기준)
- **N팀 브래킷** — 다음 2의 거듭제곱으로 패딩 후 부전승 진출, 승자 다음 라운드 연결

백엔드·DB는 사용하지 않으며, 팀원 데이터만 브라우저 `localStorage`에 저장합니다.

## 데이터 모델 (요약)

```ts
Player {
  id, nickname,
  roles: [{ position: 'tank'|'healer'|'dealer'|'random', tier: { rank, division } }, ...]
}
```

- `roles`는 **최소 1개**
- 팀 총 MMR은 보유 포지션 MMR의 **평균**을 사용
- 역할별 팀 평균은 그 포지션을 가진 멤버의 **해당 역할 티어**만 집계

## 반응형 설계

| 구간 | 대략 너비 | 레이아웃 포인트 |
|------|-----------|-----------------|
| 모바일 | ~640px 미만 | 히어로·CTA·팀짜기 버튼 풀폭, 폼 1열, 결과 카드 1열 |
| 태블릿 | 640px~1024px | 티어 입력 2열, 결과 2~3열, 브래킷 가로 스냅 스크롤 |
| 데스크톱 | 1024px 이상 | 결과 최대 4열, 넓은 히어로·여백 |

### UX 디테일

- 입력·버튼 **최소 터치 영역 약 44px**
- iOS 자동 줌 방지: 입력 `font-size: 16px` (모바일)
- 토너먼트 브래킷: **좌우 스와이프 + scroll-snap**
- `overflow-x: hidden`으로 가로 넘침 방지
- `prefers-reduced-motion` 시 애니메이션 비활성

## 사용 방법

1. **로비**에서 닉네임 입력
2. **포지션 토글**로 하나 이상 선택 (탱커/힐러/딜러/무작위)
3. 선택한 **포지션마다 티어·디비전** 입력 후 팀원 추가
4. **팀 수**를 원하는 만큼 설정 (예: 2팀 스크림, 16팀 내전 토너먼트)
5. **티어 맞춰 팀짜기** 탭/클릭
6. 결과에서 팀·포지션별 평균 MMR 확인
7. 토너먼트에서 승자 선택 → 다음 라운드 진출 · 우승 팀 표시  
   (모바일에서는 브래킷을 **좌우로 밀어** 라운드를 넘깁니다)

최소 인원: **팀 수 이상**의 팀원이 등록되어 있어야 편성할 수 있습니다.

## 티어 MMR (요약)

디비전은 **5(낮음) → 1(높음)** 입니다.

예: `브론즈 5`가 가장 낮고, `챔피언 1`이 가장 높습니다.  
내부적으로 `rankIndex * 5 + (6 - division)` 형태의 점수로 비교·배분합니다.

포지션이 여러 개면 **드래프트/역할 평균에는 그 포지션의 티어**가,  
**전체 팀 MMR에는 보유 역할 MMR 평균**이 쓰입니다.
